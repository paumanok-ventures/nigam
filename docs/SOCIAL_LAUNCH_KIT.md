### Thread 2: The Comparison

```
There are now 3 approaches to autonomous coding loops:

1. Ralph Wiggum: while true; do claude; done
2. Claude /loop: cron-style scheduling
3. Nigam: structured phases + journal memory + judges

Here's why the third one ships better code: 🧵
```

```
Ralph's insight is correct: persistence wins.

But persistence WITHOUT memory = same mistake 10 times.
Persistence WITHOUT quality gates = shipping bad code faster.
Persistence WITHOUT verification = "it works" (narrator: it didn't).

Nigam adds what Ralph was missing: structure.
```

```
PLAN → EXECUTE → VERIFY

Each phase is a separate subagent with its own context window.
No context pollution. Clean separation.

The PLAN phase applies intent verification before touching code.
The EXECUTE phase gets scored by 3 judges.
The VERIFY phase detects loops and recommends elimination.
```

```
The journal (.nigam/journal.md) is the key innovation.

Append-only. Structured. Survives context resets.
The agent reads it at every iteration start.

No more context rot.
No more forgetting what it tried.
No more "wait, didn't we fix this already?"
```
