# Nigam — Release & Zero-Budget Marketing Strategy

## The Insight

There are dozens of agentic coding loops. There is exactly ONE that roasts your code like a disappointed Indian Idol judge. `/ask-anu-malik` is the Trojan horse. People come for the roast. They stay for the loop.

## Memetic Architecture

### Primary viral vector: `/ask-anu-malik` screenshots

Every roast is unique, technically accurate, and devastatingly funny. This generates infinite screenshot content. People share the burns on X/Twitter like they share Spotify Wrapped.

**The key mechanic:** The WORSE your code is, the more entertaining the output. This is the opposite of every other tool — where bad code means a bad experience. Here, bad code = better content. People will INTENTIONALLY run it on their worst codebases for the content.

### Secondary viral vectors:

- **Golden Mic screenshots** — "My code earned a 🏆 at 3am" is inherently shareable
- **GainAm Career Level** — "I went from INTERN to STAFF" is Spotify Wrapped energy
- **Judge scores** — "Farah gave me 3/10 💀" is meme content
- **Elimination quotes** — Bollywood lyrics as error messages are screenshot gold
- **Iteration 42** — developer culture catnip
- **GainAm.com** — the mystery URL that generates curiosity clicks

### Tertiary: Cultural identity

No other dev tool has this. Ralph has a Simpsons reference (one). OpenClaw has a lobster emoji. Nigam has an ENTIRE BOLLYWOOD UNIVERSE. That's a moat of personality nobody will copy because nobody else would commit to the bit this hard.

## Phase 3: Community Flywheel (Weeks 2-8)

### The Anu Malik Roast Contribution Engine

This is the CORE growth mechanic. Here's why:

1. Person sees a devastating roast screenshot on X
2. Person installs the plugin to try it on their own code
3. Person gets roasted, screenshots it, shares it
4. Repeat

But ALSO:

5. Person thinks of a funnier roast line
6. Person submits a PR with new roast templates
7. PR gets merged → person shares "I contributed to Nigam!"
8. Repeat

The roast library grows with the community. Every new roast line is a content piece. Every contributor becomes an evangelist.

### Judge Contribution System

Each new judge is:
- A PR from a domain expert
- Named after an Indian music personality (creates cultural conversation)
- A new audience segment (Judge Vishal for security → security community discovers Nigam)
- Content: "Nigam now has a security judge named after Vishal Dadlani who reviews your code with punk rock energy"

### Translation PRs

Each translation is:
- A community from a new geography
- Content: "Nigam now speaks Tamil — featuring Ilayaraja lyrics for your midnight coding sessions"
- A new set of cultural references that resonate with a specific audience

### Weekly "Anu Malik's Greatest Hits"

Feature the community's best `/ask-anu-malik` screenshots on the project's X account. People will run it on their worst code specifically to get featured. This is intentional virality.

### Weekly "Golden Mic Awards"

Feature community Golden Mic screenshots. The opposite energy — celebrating first-try perfection. Creates aspiration alongside the roast comedy.

## Why This Will Work

1. **The roast is the moat.** Nobody will fork this to remove the personality. The personality IS the product.
2. **Screenshots are the distribution.** Every `/ask-anu-malik` run generates shareable content. No marketing budget needed when the product creates its own content.
3. **The technical depth retains.** People come for the roast, stay for the loop. CAVP, journal memory, elimination — this is real engineering.
4. **Cultural specificity creates ownership.** Indian devs will feel this is THEIRS. Non-Indian devs will find it novel. Both share for different reasons.
5. **The contribution model scales.** Roast lines, judges, translations — low-barrier PRs that create high engagement.
